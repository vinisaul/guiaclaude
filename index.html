Não é sobre ter a melhor IA. É sobre usar certo.

O mapa completo do ecossistema Claude — do chat básico aos agentes autônomos. Tudo que você precisa para escalar operações, criar sistemas inteligentes e automatizar processos do zero ao nível de produção.

  • ChatGPT + pedidos genéricos

Os 5 Pilares do Ecossistema Claude

1. Claude.ai (App Principal)

Interface Web Artefatos

O que é: Interface web para conversas naturais, upload de arquivos, análise de documentos.

  • Artefatos: Gera HTML, React, SVG, Markdown, Mermaid em tempo real — código editável ao vivo
  • Code Execution: Executa Python, cria PPTX, XLSX, DOCX, PDF direto do chat
  • Projects: Contexto persistente com até 200 arquivos (32k tokens cada)
  • Quando usar: Ideação, prototipagem, análise rápida, criação de docs do celular
  • Limitação: Sem acesso direto a APIs externas, sem automação pesada

2. Claude Console (API)

Produção Escalável

O que é: Integração programática para produção 24/7.

  • Models: Opus (melhor raciocínio), Sonnet (custo/benefício), Haiku (rápido/leve)
  • Quando usar: Chatbots em produção, processamento em massa, aplicações reais
  • Custo: Pay-as-you-go
  • Context Window: Sonnet suporta 200k tokens — processa 50 artigos sem perder contexto

3. Claude Code (CLI)

Agentic Local

O que é: Ferramenta local que executa código, manipula arquivos, testa e itera.

  • Agentic: Claude toma decisões sozinho e auto-corrige
  • Quando usar: Projetos grandes, automação local, CLIs, scripts que rodam de verdade
  • Requer: Node.js + API key
  • Como funciona: É como ter um assistente que pode executar comandos em plain text e encadear múltiplas ferramentas automaticamente

4. Skills (Prompts Especializados)

Reutilizável Compartilhável

O que são: Conjunto de instruções que Claude "aprende" e segue automaticamente.

  • Dois tipos: Claude.ai Skills (pessoal) | Claude Code Skills (equipe, versionável)
  • Valor: Economiza tokens, consistência, reutilizável
  • Estrutura: Cabeçalho YAML (name, description) + corpo Markdown
  • Ativação: Direta (/nome) ou automática
  • Argumentos dinâmicos: $ARGUMENTS

5. AIOs (Agentes Orquestrados)

Autônomo Coordenado

O que são: Múltiplos agentes com papéis diferentes trabalhando juntos, coordenados por um orquestrador.

  • Arquitetura: Agente 1 (planeja) → Agente 2 (executa) → Agente 3 (valida) → Agente N (especialista)
  • Comunicação: API, webhooks, file-based (JSON)
  • Verificação de qualidade: IA superior valida output dos agentes subordinados
  • Quando usar: Operações escaláveis, fluxos 24/7, sistemas complexos

Claude não é chat. É um computador completo no seu bolso.

Upload de PDF de 50 páginas → Análise completa com tabelas + resumo executivo. Code Execution → Criar apresentação PPTX pronta. Artefato React interativo → Dashboard funcional editável ao vivo.

Diferença entre Skills, Agentes, Plugins, MCPs e AIOs

Conceito O que é Autonomia Exemplo Quando usar
Skill Habilidade passiva — conjunto de instruções que você invoca ❌ Zero — você precisa chamar /skill Skill de code review que você ativa antes do PR Tarefas repetitivas que você executa manualmente
Agente Funcionário autônomo — recebe um to-do e executa sozinho usando skills disponíveis ✅ Alta — decide quais skills usar e quando Agente de marketing que pesquisa tendências, gera ideias e cria posts Tarefas que exigem múltiplas etapas e decisões
Plugin Extensão que adiciona capacidade nova ao sistema ❌ Zero — é uma ferramenta, não pensa Plugin Figma que exporta frames como código Adicionar funcionalidade específica
MCP Conector que liga Claude a sistemas externos (APIs, databases, ferramentas) ❌ Zero — só conecta, não decide MCP Supabase — Claude pode fazer queries SQL Integrar Claude com seus sistemas
AIOs Orquestrador de múltiplos agentes — coordena equipe de IAs com papéis diferentes ✅✅ Muito alta — gerencia agentes, valida qualidade, decide fluxos Sistema com 4 agentes: Pesquisador → Escritor → Designer → Revisor Operações complexas, produção em escala, sistemas 24/7

Configuração Avançada — Os 5 Arquivos Essenciais do Claude Code

1. CLAUDE.md (Manual do Projeto)

▼

O que faz: Instrução permanente que Claude lê antes de cada sessão

Quando usar: Definir padrões de código, arquitetura, convenções de commit

# Project Instructions

## Code Standards
- TypeScript strict mode, no `any` types
- Use @phosphor-icons/react for all icons
- Dark mode default, light mode must work

## Architecture
- Next.js 15 App Router
- Zustand for state management
- Tailwind CSS + shadcn/ui

## Commit Format
- TI-XXX: Description (YYYY-MM-DD)
- Co-Authored-By: Your Name

2. settings.json (Permissões e Ambiente)

▼

O que faz: Controla permissões de ferramentas e variáveis de ambiente

Quando usar: Whitelist de tools, bloqueio de operações perigosas, config de MCPs

{
  "permissions": {
    "allow": ["Read", "Write", "Edit", "Bash", "Glob", "Grep"],
    "deny": ["mcp__dangerous"]
  },
  "env": {
    "ANTHROPIC_MODEL": "claude-sonnet-4-20250514",
    "CLAUDE_CODE_MAX_TURNS": "100"
  }
}

3. commands/ (Comandos Customizados)

▼

O que faz: Slash commands que transformam prompts complexos em atalhos

Quando usar: Code reviews, deploys, refactors repetitivos

Exemplo: .claude/commands/review.md

---
description: Review code for bugs and security
allowed-tools: Read, Grep, Glob, Agent
---

Review the latest changes for these issues:
- Security vulnerabilities (XSS, injection, CSRF)
- Performance bottlenecks
- Best practice violations
- Missing error handling

Focus specifically on: $ARGUMENTS

Output as markdown table: File | Line | Severity | Issue

Uso: /review auth module

4. hooks (Automação Pós-Ação)

▼

O que faz: Scripts shell que rodam automaticamente antes/depois de cada tool

Quando usar: Auto-lint, auto-format, notificações

Exemplo: Dentro de settings.json

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "command": "npm run lint -- --fix --quiet"
    }],
    "PostToolUse": [{
      "matcher": "Write",
      "command": "npx prettier --write $FILE_PATH"
    }]
  }
}

5. MCP Servers (Conexões Externas)

▼

O que faz: Conecta Claude a databases, APIs, search engines

Quando usar: Query em Supabase, pesquisa web, integração GitHub

Exemplo: Dentro de settings.json

{
  "mcpServers": {
    "supabase": {
      "command": "npx -y @supabase/mcp-server",
      "args": ["--project-ref", "your-project-id"]
    },
    "brave-search": {
      "command": "npx -y @anthropic/brave-mcp",
      "env": { "BRAVE_API_KEY": "your-key" }
    },
    "github": {
      "command": "npx -y @anthropic/github-mcp",
      "env": { "GITHUB_TOKEN": "ghp_xxx" }
    }
  }
}

📊 ANÁLISE & ESTRATÉGIA

▼

1. Análise SWOT + Prioridades Estratégicas

You are a McKinsey Senior Partner with 20 years of experience in [INDUSTRY].
Conduct a full strategic SWOT analysis for [BUSINESS].

OUTPUT STRUCTURE:

1. CONTEXT & STAKES
- Key assumptions about the industry
- What changes if no action is taken in 90 days

2. SWOT MATRIX
- Strengths: 4 points + how to monetize each
- Weaknesses: 4 points + root cause
- Opportunities: 4 points + time window
- Threats: 4 points + probability and damage

3. STRATEGIC INTERSECTIONS (TOWS)
- SO: use strengths to seize opportunities
- ST: strengths defend against threats
- WO: opportunities offset weaknesses
- WT: minimize risk when weaknesses meet threats

4. TOP 5 PRIORITIES FOR NEXT 30 DAYS
For each: action | owner | metric | cost of inaction

5. EXECUTIVE SUMMARY (3 sentences)
What's happening → What's critical → What to do now

2. Análise de Concorrência

You are a BCG Strategy Director specializing in competitive intelligence for [MARKET].
Conduct deep competitive analysis for [COMPANY].

OUTPUT:

1. MARKET MAP
- Player segmentation: leaders/challengers/niche/declining
- 3 structural shifts over next 2 years

2. COMPETITIVE COMPARISON (top 5 players)
Columns: Positioning | Pricing | Features | Channel | Core weakness

3. WHERE WE WIN / LOSE
- 3 arenas with advantage — why and how long
- 3 arenas where we fall short — systemic cause

4. VALUE PROPOSITION ANALYSIS
- What competitors promise vs deliver
- Unmet pain points (white spaces)

5. 3 OUTPERFORMANCE STRATEGIES FOR 90 DAYS
Each: hypothesis → tactic → resource → KPI → kill criterion

6. EARLY WARNING SIGNALS
5 indicators a competitor is preparing aggressive move

3. Perfil de Cliente Ideal (ICP)

You are a Chief Marketing Officer with deep B2B/B2C expertise in [PRODUCT/NICHE].
Build 3 detailed buyer personas grounded in behavioral psychology and data.

STRUCTURE FOR EACH PERSONA:

— PERSONA [N]: [NAME] - [ARCHETYPE] —

1. DEMOGRAPHICS & CONTEXT
- Role, company type, life situation
- A typical day. 3 key moments when they think about our problem

2. PSYCHOGRAPHICS (deeper than demographics)
- Core fear: what keeps them awake at night
- Core desire: how life changes once the problem is solved
- Self-identity: how they describe themselves to others

3. JOBS-TO-BE-DONE
- Functional job: what they are trying to accomplish
- Emotional job: how they want to feel
- Social job: how they want to be perceived by others

4. PURCHASE TRIGGERS & BARRIERS
- 3 events that trigger the search for a solution
- 3 objections that stall the purchase
- Who influences the decision and how

5. CHANNELS & CONTENT
- Where they consume information (specific platforms, formats)
- What content converts - not engages, converts

6. THE MESSAGE THAT CLOSES THE DEAL
- One value line tailored to this persona
- The single word that must not be missing from all communication

4. Proposta de Valor

You are a Chief Strategy Officer specializing in positioning for [PRODUCT/BRAND].
Develop a compelling value proposition with zero marketing noise.

OUTPUT STRUCTURE:

1. DIAGNOSIS OF THE CURRENT STATE
- How the positioning looks today (if it exists)
- 3 main errors in the existing CVP (or typical errors in the market)

2. ANATOMY OF THE PROBLEM
- Surface pain: what the customer says
- Deep pain: what the customer feels
- Root pain: why the problem exists at all
- Cost of inaction: what the customer loses every month without a solution

3. VALUE PROPOSITION (Strategyzer framework)
- For whom: segment + context + moment
- Who want: specific, measurable outcome
- Our product: what it does
- Which: the key differentiator from alternatives
- Unlike: the main competitor or alternative

4. 3 PROOF POINTS (facts, not promises)
- Metric / case study / research + why it matters to the buyer specifically

5. TAGLINE (3 versions)
- Rational: benefit in numbers
- Emotional: identity transformation
- Provocative: attack on the status quo

6. WEBSITE HERO COPY
- H1 (max 8 words): the core promise
- H2 (max 20 words): how it works and for whom
- CTA (max 4 words): the next step with zero risk

5. Plano de Go-To-Market

You are a VP of Growth with experience launching products to [TARGET AUDIENCE].
Create a complete GTM plan for launching [PRODUCT].

OUTPUT STRUCTURE:

1. GTM STRATEGY (one-pager)
- Launch narrative: a 3-act story (problem → moment → solution)
- Motion type: PLG / SLG / hybrid - and why this one
- Launch success criterion: what must happen within 30 days

2. SEGMENTATION & ICP
- The ideal first customer: who buys without persuasion and tells others
- Segments in priority order + the logic behind the ranking

3. CHANNELS & BUDGET SPLIT
For each channel: % of budget | goal | metric | kill threshold

4. 4-WEEK CONTENT PLAN
Week 1: Build anticipation (pre-launch)
Week 2: Launch and proof
Week 3: Scale through community
Week 4: Convert and retain
For each week: 3 specific pieces of content with format and channel

5. LAUNCH KPI DASHBOARD
- Leading metrics (first 7 days)
- Lagging metrics (30 days)
- Red flags: what triggers an immediate pivot

6. WAR ROOM: PLAN B
3 failure scenarios → trigger event → response within 48 hours

1. Análise SWOT & Prioridades Estratégicas

▼

Uma análise SWOT rigorosa com cruzamento TOWS e um roteiro de prioridades para 30 dias. Não uma lista de observações, mas um plano de ação pronto para apresentar ao board.

Conduct a full strategic SWOT analysis for [BUSINESS].

- OUTPUT STRUCTURE —

1. CONTEXT & STAKES
- Key assumptions about the industry and the company's current position
- What changes if no action is taken in the next 90 days

2. SWOT MATRIX (each point is an insight, not a fact)
- Strengths: 4 points + how to monetize each one
- Weaknesses: 4 points + root cause of each
- Opportunities: 4 points + the time window to capture each
- Threats: 4 points + probability and magnitude of damage

3. STRATEGIC INTERSECTIONS (TOWS Matrix)
- SO: how to use strengths to seize opportunities
- ST: how strengths defend against threats
- WO: how opportunities can mask or offset weaknesses
- WT: risk minimization when weaknesses meet threats

4. TOP 5 PRIORITIES FOR THE NEXT 30 DAYS
For each: specific action | owner | success metric | cost of inaction

5. EXECUTIVE SUMMARY (3 sentences)
What is happening → What is critical → What to do immediately

- 3 arenas where we have an advantage - why and how long it will hold

4. VALUE PROPOSITION ANALYSIS
- What competitors promise vs. what they actually deliver

Estrutura Viral para Reels

Padrões extraídos de centenas de reels virais. Use esta estrutura exata para replicar resultados.

Elemento 1: Hook nos Primeiros 3s

  • Pattern Interrupt: Visual + áudio que quebra o scroll
  • Pergunta provocativa: "Você faz isso errado há anos?"
  • Afirmação ousada: "Isso mudou meu negócio em 7 dias"
  • Curiosity gap: Revelação parcial que gera urgência
  • Contradição: "Todo mundo diz X, mas a verdade é Y"

Elemento 2: Promessa Clara (3-10s)

  • O que você vai aprender: Específico, mensurável
  • Benefício tangível: "3 formas de dobrar seu tráfego"
  • Transformação: Estado antes → estado depois
  • Tempo: Quanto tempo leva para implementar
  • Prova social sutil: "Testei com 50 clientes"

Elemento 3: Conteúdo em Camadas (10-50s)

  • 3 passos claros: Nunca mais de 5, ideal 3
  • Ritmo rápido: Troca visual a cada 3-5 segundos
  • Texto on-screen: Reforça o que você fala
  • Exemplos concretos: Números, nomes, casos reais
  • Microtransições: Zoom, corte, movimento constante

Elemento 4: Retenção até o Final (50-60s)

  • Loop aberto: "Mas tem um segredo que ninguém fala..."
  • Tease do próximo: "No próximo vídeo te mostro como escalar isso"
  • Recap rápido: Resume em 1 frase o que foi dito
  • CTA implícito: "Salva pra não esquecer"
  • Gancho emocional final: Reforça a dor ou o desejo
